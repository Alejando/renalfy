# Implementation Plan: Receipts — Folio Generation & State Machine

**Branch**: `007-receipts-folio-states` | **Date**: 2026-03-20 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/007-receipts-folio-states/spec.md`

## Summary

Implement the Receipts module (backend only, NestJS) for Sprint 7. Core capabilities: atomic sequential folio generation per tenant+location (`{LOC}-{YYYY}-{NNNNN}`), receipt state machine (`ACTIVE → FINISHED → SETTLED | CANCELLED`), BENEFIT payment type that atomically increments plan session count and auto-exhausts plans, appointment linkage with COMPLETED-status guard, and MANAGER/STAFF location scoping. Two Prisma schema changes required: add `planId` to `Receipt` and add a new `ReceiptFolioCounter` model.

## Implementation Strategy

### Phase 1: Schema + Types (Foundation)

1. Add `planId String?` and `ReceiptFolioCounter` model to Prisma schema
2. Run `prisma migrate dev` + `prisma generate`
3. Create `packages/types/src/receipts.schemas.ts` with all Zod schemas
4. Export from `packages/types/src/index.ts`
5. Build `@repo/types`

### Phase 2: Unit Tests (Red)

Write `receipts.service.spec.ts` covering all 10 quickstart scenarios — tests must fail at this point.

### Phase 3: Service Implementation (Green)

Implement `ReceiptsService` methods:
- `create()` — folio generation in `$transaction`, appointment guard, BENEFIT plan logic
- `updateStatus()` — state machine validation
- `findAll()` — paginated list with filters + location scope
- `findOne()` — single receipt with location scope

### Phase 4: Controller + DTOs

Wire up `ReceiptsController` with 4 endpoints, `@Audit()`, `@Roles()`, `@UseGuards(JwtAuthGuard)`.

### Phase 5: Module Registration + Verification

Register `ReceiptsModule` in `app.module.ts`. Run `pnpm lint && pnpm check-types && pnpm test`.

## Key Design Decisions

### Folio Generation (atomic)

```ts
// Inside prisma.$transaction(async (tx) => { ... })
const counter = await tx.receiptFolioCounter.upsert({
  where:  { tenantId_locationId_year: { tenantId, locationId, year } },
  create: { tenantId, locationId, year, lastSequence: 1 },
  update: { lastSequence: { increment: 1 } },
});
const locationCode = location.name.slice(0, 3).toUpperCase();
const folio = `${locationCode}-${year}-${String(counter.lastSequence).padStart(5, '0')}`;
```

### State Machine

```ts
const VALID_RECEIPT_TRANSITIONS: Record<ReceiptStatus, ReceiptStatus[]> = {
  ACTIVE:    ['FINISHED', 'CANCELLED'],
  FINISHED:  ['SETTLED'],
  SETTLED:   [],
  CANCELLED: [],
};
```

Terminal states (`SETTLED`, `CANCELLED`) → throw `ConflictException` before checking transitions.

### BENEFIT + Plan atomicity

```ts
// All inside prisma.$transaction
// 1. Validate plan belongs to tenant, is ACTIVE
// 2. Create receipt with planId
// 3. Update plan: usedSessions + 1; if >= plannedSessions → status = EXHAUSTED
```

### Location scope (MANAGER/STAFF)

`userLocationId: string | null` — same pattern as `AppointmentsService`. `null` = OWNER/ADMIN (unrestricted).
